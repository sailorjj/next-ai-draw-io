# 自动布局 - 任务管理

## Story
图表生成后，点击「整形」按钮，图表自动重新排列布局，减少节点碰撞和重叠。

## Ralph-Loop 执行模式
- **迭代次数**：最多 3 次
- **每次迭代**：实现 → 单元测试 → 修复失败 → 下一轮
- **结束条件**：
  1. 所有单元测试通过（无碰撞、间距正确、属性不变）
  2. 代码编译无错误
  3. 手动测试确认按钮出现在正确位置且点击有效

## Ralph-Loop 迭代记录

### Iteration 1 - 初始实现
- ✅ 开发：重构碰撞消除算法（行分组 + 水平/垂直碰撞分离）
- ✅ 测试：16/16 通过
- 结果：单元测试通过，但真实场景布局崩溃

### Iteration 2 - 修复顶层容器碰撞问题
- **问题**：阶段 2 把容器和普通节点混在一起做碰撞检测，容器之间互相推开导致布局完全错乱
- ✅ 修复：
  - 新增 `isTopLevelContainer` 函数识别顶层容器
  - 阶段 2 分离容器和独立节点，只对独立节点做碰撞消除
  - 容器只做尺寸适配，不做位移
- ✅ 测试：16/16 通过

### Iteration 3 - 修复纵向布局被横向拉开问题
- **问题**：行分组阈值 `maxH * 0.5` 太宽松，导致 y 坐标差距较大的节点被分到同一行，纵向排列的节点被水平碰撞消除强制拉开成水平长链
- ✅ 修复：行分组阈值改为 `minH * 0.3`，只有 y 非常接近（小于最小高度的 30%）的节点才分在同一行
- ✅ 测试：16/16 通过
- ✅ 编译：TypeScript 无错误

### Iteration 4 - 修复模型输出截断检测不敏感问题
- **问题**：模型生成到一半（比如只生成了 50 个 cell），但 JSON 恰好在 `</mxCell>` 处结束，JSON 本身有效，`repairToolCall` 不会被调用，服务端截断检测完全失效
- ✅ 修复（双层面）：
  - **服务端**（`route.ts`）：降低截断检测阈值从 50 降到 10，新增 `endsAtCellBoundary` 检测
  - **客户端**（`use-diagram-tool-handlers.ts`）：新增 `isLikelyContentTruncated` 函数，即使 XML 结构完整也检测内容是否被截断（20+ cells 且结尾无任何关闭标签 = 可疑）
  - 两层检测任一命中都会提示用户用 `append_diagram` 继续
- ✅ 测试：16/16 通过
- ✅ 编译：TypeScript 无错误

## 验收标准
